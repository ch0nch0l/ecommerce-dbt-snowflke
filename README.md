## Architecture

```
┌──────────────────────────────────────────────────────────────────┐
│  SOURCE  —  Brazilian Olist E-Commerce Dataset (Kaggle)          │
│  ~100k orders · 8 tables · 2016–2018                            │
└─────────────────────────┬────────────────────────────────────────┘
                          │  dbt seed (loads CSVs to Snowflake)
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  STAGING  (views, schema: staging)                               │
│  stg_orders · stg_order_items · stg_customers · stg_products    │
│  stg_sellers · stg_order_payments · stg_order_reviews           │
│  → Rename columns, cast types, basic filters only               │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  INTERMEDIATE  (views, schema: intermediate)                     │
│  int_orders_enriched                                            │
│  → Joins all order data: customers + items + payments + reviews │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  MARTS  (tables, schema: marts)                                  │
│  fct_orders            ← core fact, incremental                 │
│  dim_customers         ← customer LTV + segmentation            │
│  dim_products          ← product catalogue + sales metrics      │
│  mart_daily_revenue    ← daily BI summary + 7d rolling avg      │
│  mart_category_performance ← revenue by product category        │
│  mart_seller_performance   ← seller tiering (top/standard/risk) │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
               Power BI / Looker Studio Dashboard
```

## Quick Start

### 1. Snowflake setup (one time)

Sign up for a [free Snowflake trial](https://signup.snowflake.com/) (30 days, no credit card).  
Run `snowflake_setup.sql` in a SQL Worksheet as ACCOUNTADMIN.

### 2. Clone and install dbt

```bash
git clone https://github.com/[yourusername]/ecommerce-dbt-snowflake.git
cd ecommerce-dbt-snowflake

python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Configure connection

```bash
cp profiles.yml.example ~/.dbt/profiles.yml
# Edit ~/.dbt/profiles.yml with your Snowflake account, user, and password
```

### 4. Install dbt packages

```bash
dbt deps
```

### 5. Run the full pipeline

```bash
# Load seed data (CSVs → Snowflake)
dbt seed

# Build all models
dbt run

# Run all tests (schema + singular)
dbt test

# Build snapshots (SCD Type 2)
dbt snapshot

# Generate and open documentation
dbt docs generate
dbt docs serve   # opens http://localhost:8080
```

### 6. Run a specific layer only

```bash
dbt run --select staging          # staging models only
dbt run --select marts            # mart models only
dbt run --select fct_orders       # single model
dbt run --select +fct_orders      # fct_orders + all upstream
dbt test --select marts           # test mart models only
```

## dbt Tests

This project includes **two types of dbt tests**:

**Schema tests** (defined in `schema.yml`):
- `not_null` — critical columns never null
- `unique` — primary keys are unique
- `accepted_values` — enums only contain valid values
- `relationships` — foreign keys resolve correctly

**Singular tests** (in `tests/`):
- `assert_positive_revenue` — no negative revenue on active orders
- `assert_no_orders_without_customers` — all orders have a customer
- `assert_review_score_range` — scores always 1–5

Run all tests: `dbt test`  
Run tests for one model: `dbt test --select fct_orders`
